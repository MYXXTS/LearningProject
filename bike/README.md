# 共享单车锁模拟项目

## 项目简介

本项目旨在模拟共享单车锁的基本功能，覆盖开锁、关锁、状态指示、结算模拟、区域限制等核心流程。

* Pin Assignment:
*   P0.0-P0.7: LCD 1602A Data Bus DB0-DB7
*   P2.5: LCD RW
*   P2.6: LCD RS
*   P2.7: LCD EN
*   P2.0: SG90 Servo PWM
*   P2.1: GPS RXD (ATGM332D-5N-GPS TXD -> MCU)
*   P2.2: GPS TXD (MCU -> ATGM332D-5N-GPS RXD)
*   P2.3: MP3 TXD (MCU -> MP3 RXD)
*   P3.0: Bluetooth RXD (HC-06 TXD -> MCU)
*   P3.1: Bluetooth TXD (MCU -> HC-06 RXD)
  
其中，GPS 使用硬件 UART，其余的硬件使用软件 UART

## 项目基本功能

1. **解锁**：用户通过扫描二维码解锁单车，单车进入使用状态。
2. **使用**：用户使用过程中，系统记录骑行时间和距离。
3. **上锁**：用户结束骑行后，手动上锁，单车进入结算状态。
4. **结算**：系统根据骑行时间和距离计算费用，并显示在 App 上。

## 硬件和软件环境

- 硬件：普中 51 单片机(STC89C52RC)、HC-06 蓝牙模块、SG90 舵机、ATGM336H-5N GPS 模块、MP3 播放模块、1602A LCD 显示屏等。
- 语言：C 语言、Flutter（用于手机 APP 开发，最低版本要求：安卓 12）
- 开发环境：Keil uVision5、Proteus 9、VS Code

## 项目结构

- `hardware/`：包含硬件连接图和相关资料。
- `firmware/`：包含单片机的 C 代码。
- `app/`：包含 Flutter App 的代码（仅保留 Android 12+ 打包所需内容）。
- `docs/`：包含项目文档和使用说明。

## 功能划分

### APP 功能

主要功能：
1. 扫描二维码解锁单车（模拟时可直接粘贴二维码文本内容）。
2. 发送开锁和关锁命令。
3. 获取 GPS 定位信息并进行区域限制判断。
4. 显示骑行时间、距离和费用。
5. 设置限制区域。
6. 模拟结算功能。

辅助功能：
1. 显示单车状态（如锁定、使用中等）。
2. 提供用户反馈（如开锁成功、失败提示等）。
3. 记录使用历史和骑行数据。
4. 提供调试后门（如临时关闭或修改区域限制功能、修改当前使用和历史使用的定位起始位置等）。
5. 错误信息面板支持一键复制错误详情到剪切板，便于排查问题。

### 单片机功能

主要功能：
1. 接收并处理来自 APP 的开锁和关锁命令。
2. 控制舵机进行开锁和关锁操作。
3. 播放提示音。
4. 更新 LCD 显示状态。
5. 响应 APP 的状态查询与提示音播放指令。

辅助功能：
1. 处理异常情况（如蓝牙通信异常等），并反馈给 APP。

> 特殊说明：由于定位模块在当前模拟场景中无法稳定获取定位与时间数据，因此单片机流程中不使用 GPS 模块。定位与时间均由手机 APP 获取并记录。定位模块在本项目中仅保持通电，不参与模拟流程。

## 流程说明

### 开锁

1. 用户通过手机 APP 扫描带有蓝牙模块 MAC 地址的二维码（模拟时可粘贴二维码内容并解析 MAC）。
2. 手机 APP 进行预判断，判断当前 GPS 定位信息是否在允许的区域内，若不在则拒绝开锁、发送开锁失败的提示音指令给单片机，让单片机播放开锁失败提示音，手机 APP 显示开锁失败的提示信息并结束流程，否则继续执行。
3. 用户确认开锁后，手机 APP 先记录当前位置信息，然后连接到蓝牙模块，发送开锁命令。
4. 单片机接收开锁命令，进行校验后执行下一步。
5. 控制舵机进行开锁操作，播放开锁提示音，更新 LCD 显示状态。
6. 单片机通过蓝牙返回开锁成功反馈。
7. 手机 APP 接收到开锁成功反馈后，使用手机本地时间和之前记录的位置信息，记录开锁时间和位置，并显示开锁成功的提示信息。
8. 进入使用状态，等待用户结束骑行并发送关锁命令。

### 关锁

1. 用户通过手机 APP 发送关锁命令。
2. 手机 APP 进行预判断，判断当前 GPS 定位信息是否在允许的区域内，若不在则拒绝关锁、发送关锁失败的提示音指令给单片机，让单片机播放关锁失败提示音，手机 APP 显示关锁失败的提示信息并结束流程，否则继续执行。
3. 用户确认关锁后，手机 APP 连接到蓝牙模块，发送关锁命令。
4. 单片机接收关锁命令，进行校验后执行下一步。
5. 控制舵机进行关锁操作，播放关锁提示音，更新 LCD 显示状态。
6. 单片机通过蓝牙返回关锁成功反馈。
7. 手机 APP 接收到关锁成功反馈后，使用手机本地时间，结合之前记录的开锁时间和位置，记录关锁时间和位置，并显示关锁成功的提示信息。
8. 手机 APP 根据开锁和关锁的时间计算使用时长，并模拟结算费用。
9. 结算后记录使用记录并显示在 APP 上。

### 设置限制区域

1. 用户通过手机 APP 直接设置限制区域，包含一系列的坐标点（使用圆形区域或多边形区域）。
2. 手机 APP 存储限制区域信息。
3. 在每次开锁或关锁时，手机 APP 获取当前 GPS 定位信息，并判断是否在限制区域内，若不在则拒绝操作并反馈给用户。

### 调试后门

1. 在手机 APP 上留一个后门，用于在测试阶段临时关闭或者修改区域限制功能、或者修改当前使用和历史使用的定位起始位置等，以便于调试和演示。

> 备注：测试的时候不会进行实际的骑行或者移动，因此调试的后门需要考虑到这一点，覆盖常见的测试场景。
