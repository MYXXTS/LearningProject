# 蓝牙通信协议文档（当前实现版）

## 1. 文档说明

本文档描述当前 `bike` 项目中，Flutter App 与单片机整机固件 `05_bike_lock_controller` 之间的真实蓝牙通信协议。

本文档以当前代码实现为准，适用于：

- Android App 端蓝牙调用
- HC-06 串口调试工具联调
- 单片机整机协议排查

## 2. 通信基础参数

| 项目 | 参数 |
| ---- | ---- |
| 蓝牙模块 | HC-06 |
| 单片机串口 | `P3.0/P3.1` 硬件 UART |
| 波特率 | `9600` |
| 数据格式 | `8N1` |
| 编码 | UTF-8 文本，命令内容使用 ASCII 子集 |
| 推荐交互方式 | 一问一答 |

说明：

- App 通过经典蓝牙 SPP 与 HC-06 通信。
- 单片机接收文本命令并返回文本响应。
- 当前协议不使用二进制帧，不使用 CRC。

## 3. 职责划分

### 3.1 App 负责

- 扫码与 MAC 解析
- 蓝牙连接
- 手机定位获取
- 围栏设置与围栏判断
- 开锁/关锁前置判断
- 时间记录、骑行记录、费用计算

### 3.2 单片机负责

- 接收 `U/L/S/P,n` 指令
- 控制舵机开锁/关锁
- 播放 MP3 提示音
- 更新 LCD 显示
- 返回协议结果

### 3.3 GPS 说明

当前整机流程中：

- GPS 模块仍保留接线
- 不参与授时
- 不参与定位
- 不参与协议

## 4. 命令格式

所有命令都是文本命令，格式如下：

```text
<命令字>[,参数]
```

支持的结束方式：

- `\n`
- `\r`
- `\r\n`

另外，当前固件还支持以下“无换行立即识别”模式：

- `U`
- `L`
- `S`
- `P,<单个数字>`

例如：

```text
U
L
S
P,1
```

也能被当前固件直接识别。

## 5. 大小写规则

当前固件对命令字母大小写不敏感：

- `U` / `u`
- `L` / `l`
- `S` / `s`
- `P,1` / `p,1`

都可以被识别。

## 6. 下行命令（App -> 单片机）

### 6.1 开锁

```text
U
```

推荐发送：

```text
U\n
```

### 6.2 关锁

```text
L
```

推荐发送：

```text
L\n
```

### 6.3 查询状态

```text
S
```

推荐发送：

```text
S\n
```

### 6.4 播放提示音

```text
P,<n>
```

其中：

- `n=1..5` 为有效曲目编号
- `n=0` 或 `n>5` 会被判定为非法编号

示例：

```text
P,1\n
P,5\n
```

## 7. 上行响应（单片机 -> App）

### 7.1 开锁响应

| 响应 | 含义 |
| ---- | ---- |
| `U,OK` | 开锁成功 |
| `U,ALREADY` | 当前已经处于解锁状态 |
| `U,BUSY` | 当前正在执行动作，暂时无法处理新的开锁请求 |

### 7.2 关锁响应

| 响应 | 含义 |
| ---- | ---- |
| `L,OK` | 关锁成功 |
| `L,ALREADY` | 当前已经处于锁定状态 |
| `L,BUSY` | 当前正在执行动作，暂时无法处理新的关锁请求 |

### 7.3 状态响应

| 响应 | 含义 |
| ---- | ---- |
| `S,0` | 当前锁定 |
| `S,1` | 当前解锁/骑行中 |

### 7.4 播放提示音响应

| 响应 | 含义 |
| ---- | ---- |
| `P,OK` | 播放命令已接收并开始执行 |
| `P,BAD` | 曲目编号非法 |
| `P,BUSY` | 当前处于忙状态，不能立即处理播放请求 |

### 7.5 非法命令响应

| 响应 | 含义 |
| ---- | ---- |
| `E,BAD` | 不支持的命令文本 |

## 8. 曲目映射

| `P,n` 参数 | 对应曲目 | 典型用途 |
| ---- | ---- | ---- |
| `P,1` | `001` | 开锁成功提示音 |
| `P,2` | `002` | 开锁重复/失败提示音 |
| `P,3` | `003` | 关锁成功提示音 |
| `P,4` | `004` | 关锁重复/失败提示音 |
| `P,5` | `005` | 围栏拒绝提示音 |

## 9. 状态机与忙状态说明

单片机内部只有两个主状态：

- `LOCKED`
- `UNLOCKED`

开锁或关锁执行期间会短暂进入忙状态：

- LCD 第 1 行会显示 `STATE: BUSY`
- 此时 `U/L/P` 会返回对应的 `BUSY`
- `S` 仍然允许查询

## 10. App 侧推荐流程

### 10.1 开锁

1. App 获取手机定位
2. App 做围栏判断
3. 围栏不通过：
   - 不发送 `U`
   - 可发送 `P,5`
4. 围栏通过：
   - 发送 `U`
   - 等待 `U,OK` / `U,ALREADY` / `U,BUSY`

### 10.2 关锁

1. App 获取手机定位
2. App 做围栏判断
3. 围栏不通过：
   - 不发送 `L`
   - 可发送 `P,5`
4. 围栏通过：
   - 发送 `L`
   - 等待 `L,OK` / `L,ALREADY` / `L,BUSY`

### 10.3 状态查询

```text
S\n
```

推荐在以下场景使用：

- 初次连接后同步状态
- 发现本地骑行记录与单片机状态不一致时恢复状态

## 11. 超时建议

| 命令 | 建议超时 |
| ---- | ---- |
| `S` | 3 秒 |
| `P,n` | 3 秒 |
| `U` | 15 秒 |
| `L` | 15 秒 |

## 12. 调试建议

### 12.1 使用蓝牙串口工具测试

推荐直接发送：

```text
S
U
L
P,1
P,5
```

### 12.2 非法命令示例

以下内容会返回 `E,BAD` 或 `P,BAD`：

```text
HELLO
OPEN
P,9
P,12
```

其中：

- `HELLO`、`OPEN` 属于不支持文本，通常返回 `E,BAD`
- `P,9` 属于格式正确但编号非法，返回 `P,BAD`
- `P,12` 不符合当前固定长度解析规则，不应作为有效命令使用

## 13. 实现边界

- 当前协议不支持一次发送多条命令拼包。
- 当前协议不支持中文命令。
- 当前协议不支持围栏参数下发。
- 单片机不返回定位、时间、费用等信息。
